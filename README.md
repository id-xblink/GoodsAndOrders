# Goods And Orders


## Настройка БД
За стартовые данные отвечают последние 4 миграции
> Если миграции не проходят, то может помочь удаление бд по запрашиваемому пути (адресу)

Если бд будет развёрнута в докер контейнере, то можно воспользоваться командой ниже:
```powershell
docker run -d --name postgres -e POSTGRES_USER=postgres -e POSTGRES_PASSWORD=admin -e POSTGRES_DB=goods_and_orders_db -p 5454:5432 postgres
```

- Для backend: смотреть файл appsettings.json чтобы найти строку подключения и при необходимости изменить
(на данный момент Host=localhost;Port=5454;Database=goods_and_orders_db;Username=postgres;Password=admin)

- Для frontend: смотреть файл .env чтобы вставить строку подключения к backend
(на данный момент https://localhost:7212/api)

## Вход в систему
Пользователи для тестирования:
| Логин | Пароль | Роль |
| ------ | ------ | ------ |
| admin | admin | Админ |
| manager | manager | Менеджер |
| customer | customer | Заказчик |

## Функции приложения (ролей)
Пользователь:
- Авторизация и регистрация (только как заказчик)

Админ и менеджер:
- Просмотреть, создавать, редактировать и удалять пользователей, товары и заказы, но заказы удалять нельзя

Заказчик:
- Просмотр товаров
- Набирать, редактировать, оформлять и удалять корзину товаров
- Оформление (создание) заказа на основе корзины товаров
- Удалять свои заказы
